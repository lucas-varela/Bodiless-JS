### `Const` withDesignAt

▸ **withDesignAt**‹**C**›(...`paths`: [DesignPath](globals.md#designpath)[]): *(Anonymous function)*

*Defined in [packages/fclasses/src/withDesignAt.ts:89](https://github.com/lucas-varela/Bodiless-JS/blob/c57f63f7/packages/fclasses/src/withDesignAt.ts#L89)*

Applies a token or design at a particular path in a component with nested
designable elements.

**`example`** 
```js
withDesignAt(['A'], ['A', 'B'])(myToken)
```
is the same as:
```js
asToken(
  withDesign({
    A: myToken,
  }),
  withDesign({
    A: withDesign({
      B: myToken,
    }),
  }),
);
```
Or, you can use a shorthand overload to specify a design to be applied
rather than a token.
```js
withDesignAt(['A'])(myDesign, myTokenMeta)
```
is the same as
```
withDesignAt(['A'])(
  asToken(withDesign(myDesign), myTokenMeta)
)
```
Here, `myDesign` is a design object, not a function, eg:
```js
const myDesign = {
  Foo: addClasses('foo'),
  Bar: addClasses('bar'),
}
```

**Type parameters:**

▪ **C**: *[DesignableComponents](globals.md#designablecomponents)*

**Parameters:**

Name | Type | Description |
------ | ------ | ------ |
`...paths` | [DesignPath](globals.md#designpath)[] |  An array of paths, each of which is itself an array of strings representing a set of nested design keys.  |

**Returns:** *(Anonymous function)*

A function which accepts a design or a token, and returns a token which will
apply that design or token to all matching paths.

### `Const` withoutProps

▸ **withoutProps**‹**A**›(`keys`: keyof A | keyof A[], ...`restKeys`: keyof A[]): *[HOC](globals.md#hoc)‹object, Partial‹A››*

*Defined in [packages/fclasses/src/hoc-util.tsx:92](https://github.com/lucas-varela/Bodiless-JS/blob/c57f63f7/packages/fclasses/src/hoc-util.tsx#L92)*

Removes the specified props before rendering the wrapped component.

**Type parameters:**

▪ **A**: *object*

**Parameters:**

Name | Type |
------ | ------ |
`keys` | keyof A &#124; keyof A[] |
`...restKeys` | keyof A[] |

**Returns:** *[HOC](globals.md#hoc)‹object, Partial‹A››*

An HOC which accepts a component and returns another which accepts the
specified props and removes them before rendering.

Note this expects a type parameter specifying the props which will be
removed. For example
```
type Foo = {
  A: string,
};
type Bar = {
  B: string,
  C: String,
};
const F:FC<Foo> = () => <></>;
const G = withoutProps('B', 'C')(F);
  // ComponentWithMeta<Pick<Foo, "A"> & Partial<{ B: any } & { C: any }>>
const H = withoutProps<Bar>('B', 'C')(F); // ComponentWithMeta<Pick<Foo, "A"> & Bar>
const I = withoutProps<Bar>('B', 'D')(F); // expected type error: D not in keyof Bar)
const J = withoutProps<Bar>('X')(F); // expected type error (X not in keyof Bar)
```
